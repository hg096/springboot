
http 요청 방식
get : 데이터 요청 - select
post : 데이터 추가 - insert
put : 데이터 수정 - update
delete : 데이터 삭제 - delete 

-statefull방식(http)
연결선을 끊지 않고 계속 연결

-stateless방식(http)
요청마다 응답하고 연결선을 끊음
요청마다 끊기때문에 서버의 부하가 줄어듬

*mime 타입
타입/상세타입 형식


*http 통신 
-패킷 스위칭
하나의 선으로 데이터를 전송 
받는 서버에서는 섞인 데이터를 분류 
전송시에 순서가 꼬이지않게 데이터의 순서와 보낸곳의 데이터를 함께 전송

-서킷 스위칭
데이터를 교환할때 뭉쳐서 전송
선을 많이 연결해서 많은 비용이 발생
